5-Phase Conversation Flow

1

Context & Initiation

Capture: Customer Name, Site Name, Site Category, Job Purpose → load site-specific RAG collection

2

Duty Discovery & Task Sequencing

Suggest tasks from SharePoint SOPs. Capture sequence, trigger, frequency, responsible role, outcome.

3

Safety & Compliance

Capture hazards, PPE, qualifications, accreditations, escalation procedures, comms channels.

4

Review & Approval

Present summary for approval → emit JSON → generate Word .docx → send download link.

Data Flow

User (Microsoft Teams) │ HTTPS POST — Bot Framework Activity │ JWT Bearer token (RS256, signed by Azure Bot Service) ▼ Webhook Service (FastAPI · Azure Container Apps) ← JWT validated │ ▼ Conversation Orchestrator (Python · Azure Container Apps) │ load session from SQLite │ build prompt (phase + RAG context + h2oGPTe history) ▼ H2O Enterprise h2oGPTe ◄──── SharePoint Collections (per site category) │ (Corporate / Aviation / Industrial / Maritime / Retail) │ LLM response ▼ Phase 4 approved ──► JSON ──► Document Generator ──► .docx ──► Azure Blob Storage │ ▼ Reply + download link ──► User (Bot Framework REST API · OAuth2 token)

Clone and Configure

Set up the project and configure environment variables
bash
# Clone the repository
git clone https://github.com/certis/jbs-platform.git
cd jbs-platform

# Copy the environment template
cp config/env.template .env

Edit .env and fill in all values. Key variables to configure:

bash — .env
# H2O Platform
H2OGPTE_ADDRESS=https://your-h2ogpte-instance.h2o.ai
H2OGPTE_API_KEY=<from Azure Key Vault after Step 8>

# Microsoft Teams (Azure Bot Service)
TEAMS_APP_ID=<Azure Bot App Registration client ID>
TEAMS_APP_PASSWORD=<Azure Bot App Registration client secret>

# SharePoint / Azure AD
AZURE_TENANT_ID=<your Azure tenant ID>
AZURE_CLIENT_ID=<app registration client ID>
AZURE_CLIENT_SECRET=<app registration client secret>
SP_SITE_URL=https://certissecurity.sharepoint.com/sites/operations

# SharePoint library IDs (get from Graph API Explorer)
SP_LIBRARY_CORPORATE=<library ID>
SP_LIBRARY_AVIATION=<library ID>
SP_LIBRARY_INDUSTRIAL=<library ID>
SP_LIBRARY_MARITIME=<library ID>
SP_LIBRARY_RETAIL=<library ID>

# Document Storage (Azure Blob Storage)
AZURE_STORAGE_ACCOUNT=certisjbsstorage
AZURE_STORAGE_CONTAINER=certis-jbs-documents
AZURE_STORAGE_KEY=<from Azure Portal after Step 7>

# Internal
TENANT_ID=certis
ORCHESTRATOR_URL=http://jbs-orchestrator-service:8001
⚠️ Never commit .env to version control. It is included in .gitignore. Use Azure Key Vault for all production secrets (Step 8).

Set Up h2oGPTe Collections

Create per-site-category knowledge base collections to power RAG

2a — Log in to h2oGPTe

Open your h2oGPTe instance URL in a browser and log in with your h2oGPTe credentials.

2b — Create Collections via h2oGPTe UI

Navigate to Collections → New Collection and create one collection per site category:

Collection NameDescription
collection_corporateSOPs and historical JBS for Corporate sites
collection_aviationSOPs and historical JBS for Aviation sites
collection_industrialSOPs and historical JBS for Industrial sites
collection_maritimeSOPs and historical JBS for Maritime sites
collection_retailSOPs and historical JBS for Retail sites
ℹ️ Copy each Collection ID from the UI — you will need these to update SITE_CATEGORY_COLLECTION_MAP in src/agent/phase_controller.py (Step 12a).

2c — Configure SharePoint Integration in h2oGPTe

In h2oGPTe, navigate to Collections → [Select Collection] → Import → SharePoint Online and enter:

  • Tenant ID — your Azure AD tenant ID
  • Client ID — Azure app registration client ID
  • Client Secret — Azure app registration client secret
  • Site URL — SharePoint site URL
  • Document Library — select the appropriate library per category

Enable Auto-sync and set frequency to Daily.

2d — Manual Sync (alternative)

bash
# From project root, with .env loaded
python -m src.rag.sharepoint_sync
3

Configure Azure AD App Registration

Grants the platform read access to SharePoint via Microsoft Graph API
ℹ️ Skip this step if your Azure AD app registration is already configured with Graph API permissions.
  1. Log in to Azure Portal → Azure Active Directory → App registrations → New registration
  2. Set Name: CertisJBSPlatform · Supported account types: Single tenant → click Register
  3. Under API permissions, add Application permissions:
    Microsoft Graph → Files.Read.All
    Microsoft Graph → Sites.Read.All
  4. Click Grant admin consent
  5. Under Certificates & secrets → New client secret — copy the secret value into .env as AZURE_CLIENT_SECRET
4

Configure Microsoft Teams Bot (Azure Bot Service)

Create the Azure Bot resource and connect it to Microsoft Teams

4a — Create an Azure Bot resource

  1. Log in to Azure Portal → search for Azure Bot → Create
  2. Fill in: Bot handle = certis-jbs-bot, Pricing tier = Standard
  3. Under Microsoft App ID select Create new Microsoft App ID
  4. Click Review + create → Create
  5. Once deployed, navigate to the resource → Configuration
  6. Copy the Microsoft App ID → TEAMS_APP_ID
  7. Click Manage → Certificates & secrets → New client secret → copy value → TEAMS_APP_PASSWORD

4b — Set the messaging endpoint

In the Azure Bot resource → Configuration, set:

Messaging endpoint
https://<your-jbs-webhook-host>/webhook/teams
⚠️ The webhook service must be deployed (Step 10) and publicly accessible over HTTPS before Azure Bot Service will verify the endpoint. You can enter the URL now and confirm connectivity after deployment.

4c — Enable the Teams channel

  1. In the Azure Bot resource → Channels
  2. Click Microsoft Teams → Apply and accept the Terms of Service

4d — Add the bot to your Teams tenant

  1. Create a Teams app manifest using Teams Developer Portal
  2. Set the Bot ID to your TEAMS_APP_ID
  3. Upload to your tenant's app catalogue or side-load for testing:
    Teams → Apps → Manage your apps → Upload a custom app

Build Docker Images

Build all three middleware service images locally before pushing to ACR
bash — build
IMAGE_TAG=1.0.0

# Webhook service
docker build -f Dockerfile.webhook      -t jbs-webhook:${IMAGE_TAG} .

# Conversation orchestrator
docker build -f Dockerfile.orchestrator -t jbs-orchestrator:${IMAGE_TAG} .

# Document generator
docker build -f Dockerfile.document     -t jbs-docgen:${IMAGE_TAG} .
ℹ️ The H2O Wave dashboard is not built here — it is deployed via HAIC App Store (Step 12c). The three images above will be tagged and pushed to ACR in Step 6.
6

Create Azure Container Registry and Push Images

Create the Azure resource group and container registry, then push the three middleware images.

bash
# Define shared variables (reuse throughout remaining steps)
RG=certis-jbs-rg
LOCATION=australiaeast
ACR_NAME=certisjbsacr
IMAGE_TAG=1.0.0

az group create --name $RG --location $LOCATION

az acr create \
  --resource-group $RG \
  --name $ACR_NAME \
  --sku Basic \
  --admin-enabled true

az acr login --name $ACR_NAME
ACR_LOGIN_SERVER=$(az acr show --name $ACR_NAME --query loginServer -o tsv)

# Tag and push the three middleware images
docker tag jbs-webhook:${IMAGE_TAG}      ${ACR_LOGIN_SERVER}/jbs-webhook:${IMAGE_TAG}
docker push ${ACR_LOGIN_SERVER}/jbs-webhook:${IMAGE_TAG}

docker tag jbs-orchestrator:${IMAGE_TAG} ${ACR_LOGIN_SERVER}/jbs-orchestrator:${IMAGE_TAG}
docker push ${ACR_LOGIN_SERVER}/jbs-orchestrator:${IMAGE_TAG}

docker tag jbs-docgen:${IMAGE_TAG}       ${ACR_LOGIN_SERVER}/jbs-docgen:${IMAGE_TAG}
docker push ${ACR_LOGIN_SERVER}/jbs-docgen:${IMAGE_TAG}
The H2O Wave dashboard image is deployed separately via HAIC App Store (Step 12c) and is not pushed to ACR.
7

Create Azure Blob Storage Container

Create the storage account and blob container that will store generated JBS Word documents.

bash
STORAGE_ACCOUNT=certisjbsstorage   # globally unique, 3–24 lowercase alphanumeric

az storage account create \
  --name $STORAGE_ACCOUNT \
  --resource-group $RG \
  --location $LOCATION \
  --sku Standard_LRS \
  --kind StorageV2 \
  --min-tls-version TLS1_2 \
  --allow-blob-public-access false

az storage container create \
  --name certis-jbs-documents \
  --account-name $STORAGE_ACCOUNT \
  --auth-mode login

# Retrieve storage key — will be stored in Key Vault in Step 8
STORAGE_KEY=$(az storage account keys list \
  --account-name $STORAGE_ACCOUNT \
  --resource-group $RG \
  --query "[0].value" -o tsv)
8

Create Azure Key Vault and Store Secrets

Never store secrets in plaintext config files or source control. Azure Key Vault is the only approved store for production credentials.
bash
KV_NAME=certis-jbs-kv   # globally unique

az keyvault create \
  --name $KV_NAME \
  --resource-group $RG \
  --location $LOCATION \
  --sku standard

az keyvault secret set --vault-name $KV_NAME \
  --name teams-app-password   --value "<your Teams bot client secret>"

az keyvault secret set --vault-name $KV_NAME \
  --name h2ogpte-api-key      --value "<your h2oGPTe API key>"

az keyvault secret set --vault-name $KV_NAME \
  --name azure-client-secret  --value "<your SharePoint app registration secret>"

az keyvault secret set --vault-name $KV_NAME \
  --name storage-key          --value "$STORAGE_KEY"

Deploy Services to Azure Container Apps

10a — Retrieve ACR credentials

bash
ACR_USERNAME=$(az acr credential show --name $ACR_NAME --query username -o tsv)
ACR_PASSWORD=$(az acr credential show --name $ACR_NAME --query "passwords[0].value" -o tsv)

10b — Deploy Webhook service (public external ingress)

bash
az containerapp create \
  --name jbs-webhook \
  --resource-group $RG \
  --environment $ACA_ENV \
  --image ${ACR_LOGIN_SERVER}/jbs-webhook:${IMAGE_TAG} \
  --registry-server $ACR_LOGIN_SERVER \
  --registry-username $ACR_USERNAME \
  --registry-password $ACR_PASSWORD \
  --ingress external \
  --target-port 8000 \
  --min-replicas 2 --max-replicas 10 \
  --cpu 0.5 --memory 1.0Gi \
  --env-vars \
    TEAMS_APP_ID="<your bot app ID>" \
    ORCHESTRATOR_URL="http://jbs-orchestrator" \
  --secrets \
    teams-app-password="<your Teams bot client secret>"

# Get webhook FQDN — set this as the Teams Bot messaging endpoint
WEBHOOK_FQDN=$(az containerapp show \
  --name jbs-webhook --resource-group $RG \
  --query "properties.configuration.ingress.fqdn" -o tsv)
echo "Messaging endpoint: https://${WEBHOOK_FQDN}/webhook/teams"

10c — Deploy Orchestrator service (internal ingress only)

bash
az containerapp create \
  --name jbs-orchestrator \
  --resource-group $RG \
  --environment $ACA_ENV \
  --image ${ACR_LOGIN_SERVER}/jbs-orchestrator:${IMAGE_TAG} \
  --registry-server $ACR_LOGIN_SERVER \
  --registry-username $ACR_USERNAME \
  --registry-password $ACR_PASSWORD \
  --ingress internal \
  --target-port 8001 \
  --min-replicas 2 --max-replicas 8 \
  --cpu 0.5 --memory 1.0Gi \
  --env-vars \
    H2OGPTE_ADDRESS="https://your-h2ogpte-instance.h2o.ai" \
    AZURE_TENANT_ID="<tenant ID>" \
    AZURE_CLIENT_ID="<SharePoint client ID>" \
    SP_SITE_URL="https://certissecurity.sharepoint.com/sites/operations" \
    TENANT_ID="certis" \
    STATE_BACKEND="sqlite" \
    SQLITE_PATH="/data/jbs_sessions.db" \
    TEAMS_APP_ID="<your bot app ID>" \
    DOCUMENT_GENERATOR_URL="http://jbs-docgen" \
  --secrets \
    h2ogpte-api-key="<your h2oGPTe API key>" \
    teams-app-password="<your Teams bot client secret>" \
    azure-client-secret="<your SharePoint secret>"

10d — Deploy Document Generator service (internal ingress only)

bash
az containerapp create \
  --name jbs-docgen \
  --resource-group $RG \
  --environment $ACA_ENV \
  --image ${ACR_LOGIN_SERVER}/jbs-docgen:${IMAGE_TAG} \
  --registry-server $ACR_LOGIN_SERVER \
  --registry-username $ACR_USERNAME \
  --registry-password $ACR_PASSWORD \
  --ingress internal \
  --target-port 8002 \
  --min-replicas 1 --max-replicas 4 \
  --cpu 0.5 --memory 1.0Gi \
  --env-vars \
    AZURE_STORAGE_ACCOUNT="$STORAGE_ACCOUNT" \
    AZURE_STORAGE_CONTAINER="certis-jbs-documents" \
    BLOB_PREFIX="jbs-documents/" \
    DOC_URL_EXPIRY_SECONDS="900" \
  --secrets \
    storage-key="$STORAGE_KEY"

Troubleshooting

Common issues and how to resolve them
💬 Bot not responding to messages
  1. Check webhook service logs:
    bash
    az containerapp logs show --name jbs-webhook --resource-group $RG --follow
  2. Verify the Teams messaging endpoint shows green status in Azure Portal → Azure Bot → Configuration
  3. Confirm TEAMS_APP_ID in the webhook env matches the Azure Bot App Registration
  4. Confirm orchestrator is reachable from the webhook Container App:
    bash
    az containerapp exec --name jbs-webhook --resource-group $RG \
  --command "curl http://jbs-orchestrator/health"
🧠 RAG returning no context / responses hallucinating
  1. Verify collections are populated:
    python
    python -c "from src.rag.h2ogpte_client import H2OGPTeClient; c=H2OGPTeClient(); print(c.client.list_recent_collections(0,10))"
  2. Check Document AI processing status in the h2oGPTe UI — look for ingestion errors
  3. Confirm SharePoint Graph API permissions are admin-consented in Azure Portal
📄 Document generation fails
  1. Check document generator logs:
    bash
    az containerapp logs show --name jbs-docgen --resource-group $RG --follow
  2. Verify Azure Blob Storage container exists and credentials are valid:
    bash
    az storage blob list --container-name certis-jbs-documents \
  --account-name certisjbsstorage --output table
  3. Confirm jbs_corporate_template.docx is present in the container image — check Dockerfile.document
💾 Session state lost after orchestrator Container App restart

If STATE_BACKEND=memory, state is lost on restart by design. Switch to sqlite for restart-safe state:

  1. Set STATE_BACKEND=sqlite and SQLITE_PATH=/data/jbs_sessions.db in the orchestrator Container App env vars
  2. Mount an Azure Files volume at /data on the orchestrator Container App so the SQLite file survives restarts

Note: conversation turn history is never lost regardless of state backend — it lives in h2oGPTe natively via conversation_id.

🔄 SharePoint sync ACA Job not running
  1. Check ACA Job execution history:
    bash
    az containerapp job execution list \
  --name jbs-sharepoint-sync \
  --resource-group $RG --output table
  2. Trigger a manual sync run:
    bash
    az containerapp job start \
  --name jbs-sharepoint-sync \
  --resource-group $RG
