Workflow Steps

Click a step to highlight nodes & edges on the diagram
1

User Sends Message

Security personnel send a message in Microsoft Teams to initiate or continue a JBS interview session. Azure Bot Service delivers the event as a Bot Framework Activity.

2

Webhook Validates JWT Token

Webhook Service receives the Bot Framework Activity and validates the RS256 JWT Bearer token issued by Azure Bot Service, using the Bot Framework JWKS endpoint to prevent spoofing.

3

Forward to Orchestrator

Validated payload normalised to internal schema (channel, user_id, text, reply_to) and POSTed to the Conversation Orchestrator service.

4

Session Loaded & Phase Determined

Orchestrator loads session from in-process memory (MemoryStateBackend, deepcopy isolation). Phase 1 is a code-only 5-step counter collecting customer, site, category, purpose, and authorized_by — no LLM calls. Phase 2 is hybrid LLM+RAG.

5

Prompt Built & h2oGPTe Called (Phase 2 only)

Phase 2 section prompt assembled with anti-hallucination rules. Sent to h2oGPTe with conversation_id=None — each section (duties, tasks, safety) gets a fresh RAG query, not a multi-turn conversation. Vector search runs across the site-category collection.

6

RAG Vector Search + LLM Response

h2oGPTe retrieves relevant SOP chunks from the correct collection (Corporate / Aviation / Industrial / Maritime / Retail) and generates a grounded response via hnswlib vector search.

7

Response Returned to User

Orchestrator saves updated phase state to in-process MemoryStateBackend (deepcopy ensures session isolation). LLM reply is sent directly to Teams via the Bot Framework REST API, using an OAuth2 access token obtained from Azure AD (client credentials, cached in-process).

8

Phase 2: Review & Approval

After duties+tasks and safety sections are confirmed, Orchestrator presents a structured JBS summary and awaits approval. On APPROVE, the complete JBS JSON payload is assembled and sent to the Document Generator service.

9

.docx Rendered & Uploaded to Azure Blob

python-docx renders JBS JSON into the corporate Word template. File uploaded to Azure Blob Storage. A 15-minute SAS token download URL is sent back to the user.

11

Daily SharePoint Sync

CronJob runs at 02:00 UTC pulling changed SOP documents from SharePoint via Microsoft Graph API. H2O Document AI processes (OCR, chunking) and ingests into the correct h2oGPTe site-category collection.

12

Admin Monitoring & Control

H2O Wave Admin Dashboard (HAIC App Store native) monitors active interviews, document library, and triggers SharePoint syncs. H2O MLOps tracks inference latency, error rates, and drift.
