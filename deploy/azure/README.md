# Azure Infrastructure — Bicep Template

`main.bicep` provisions the complete Azure infrastructure for the JBS Automation Platform in a single deployment command.

## Resources provisioned

| Resource | Purpose |
|---|---|
| Log Analytics Workspace | Required by Container Apps Environment |
| Azure Storage Account + Blob container | Document storage (generated .docx files) |
| Azure Key Vault | Secrets management |
| Container Apps Environment | Hosts all three middleware services |
| Webhook Container App | Public HTTPS ingress (Teams Bot) |
| Orchestrator Container App | Internal ingress (phase state machine) |
| Document Generator Container App | Internal ingress (.docx rendering) |
| SharePoint Sync Scheduled Job | Daily ACA Job (02:00 UTC) |

## Prerequisites

1. Azure CLI v2.55+ with `containerapp` extension
2. Docker images already built and pushed to ACR (Steps 5–6 in DEPLOYMENT_GUIDE.md)
3. Resource group already created:
   ```bash
   az group create --name jbs-rg --location australiaeast
   ```

## Usage

### Option A — Inline parameters

```bash
ACR_LOGIN_SERVER=jbsacr.azurecr.io
ACR_USERNAME=$(az acr credential show --name jbsacr --query username -o tsv)
ACR_PASSWORD=$(az acr credential show --name jbsacr --query "passwords[0].value" -o tsv)

az deployment group create \
  --resource-group jbs-rg \
  --template-file deploy/azure/main.bicep \
  --parameters \
    prefix=jbs \
    imageTag=1.0.0 \
    acrLoginServer=$ACR_LOGIN_SERVER \
    acrUsername=$ACR_USERNAME \
    acrPassword=$ACR_PASSWORD \
    teamsAppId="<your-teams-app-id>" \
    teamsAppPassword="<your-teams-app-password>" \
    h2ogpteApiKey="<your-h2ogpte-api-key>" \
    h2ogpteAddress="https://your-h2ogpte-instance.h2o.ai" \
    azureClientId="<sharepoint-client-id>" \
    azureClientSecret="<sharepoint-client-secret>" \
    spSiteUrl="https://wigyoai.sharepoint.com/sites/h2O" \
    spLibraryCorporate="<id>" \
    spLibraryAviation="<id>" \
    spLibraryIndustrial="<id>" \
    spLibraryMaritime="<id>" \
    spLibraryRetail="<id>"
```

### Option B — Parameters file

Create a `deploy/azure/parameters.json` file (do NOT commit this file — it contains secrets):

```json
{
  "$schema": "https://schema.management.azure.com/schemas/2019-04-01/deploymentParameters.json#",
  "contentVersion": "1.0.0.0",
  "parameters": {
    "prefix":             { "value": "jbs" },
    "imageTag":           { "value": "1.0.0" },
    "acrLoginServer":     { "value": "jbsacr.azurecr.io" },
    "acrUsername":        { "value": "<acr-username>" },
    "acrPassword":        { "value": "<acr-password>" },
    "teamsAppId":         { "value": "<teams-app-id>" },
    "teamsAppPassword":   { "value": "<teams-app-password>" },
    "h2ogpteApiKey":      { "value": "<h2ogpte-api-key>" },
    "h2ogpteAddress":     { "value": "https://your-h2ogpte-instance.h2o.ai" },
    "azureClientId":      { "value": "<sharepoint-client-id>" },
    "azureClientSecret":  { "value": "<sharepoint-client-secret>" },
    "spSiteUrl":          { "value": "https://wigyoai.sharepoint.com/sites/h2O" },
    "spLibraryCorporate": { "value": "<id>" },
    "spLibraryAviation":  { "value": "<id>" },
    "spLibraryIndustrial":{ "value": "<id>" },
    "spLibraryMaritime":  { "value": "<id>" },
    "spLibraryRetail":    { "value": "<id>" }
  }
}
```

Then deploy:
```bash
az deployment group create \
  --resource-group jbs-rg \
  --template-file deploy/azure/main.bicep \
  --parameters @deploy/azure/parameters.json
```

## Outputs

After deployment, retrieve the webhook messaging endpoint:

```bash
az deployment group show \
  --resource-group jbs-rg \
  --name main \
  --query properties.outputs.webhookMessagingEndpoint.value -o tsv
```

Set this URL in **Azure Portal → Azure Bot → Configuration → Messaging endpoint**.

## Updating an existing deployment

To update images after a new build:

```bash
az deployment group create \
  --resource-group jbs-rg \
  --template-file deploy/azure/main.bicep \
  --parameters @deploy/azure/parameters.json \
  --parameters imageTag=1.1.0
```

## Notes

- `parameters.json` is listed in `.gitignore` — never commit it as it contains secrets.
- The Azure Key Vault is created with RBAC authorization. Assign the `Key Vault Secrets User` role to any managed identity or user that needs to read secrets.
- The Storage Account has public blob access disabled. Documents are accessed only via SAS token URLs generated by the Document Generator service.
